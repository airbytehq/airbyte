# feat(docs): Add public API reference documentation with curated tags and custom schema components

## What

Adds autogenerated public API reference documentation to the Airbyte docs site. The docs are built from the public API OpenAPI spec, filtered to only include curated public endpoints, and enhanced with custom React components for source configuration and response schemas.

Key capabilities:
- Fetches and merges 17 individual OpenAPI spec files from `airbyte-platform` into a single spec at build time
- Curated tag system (`public_api_tags.json`) controls which endpoints appear — 56 operations across 15 categories
- Custom `SourceRequestSchema` component with searchable source selector and per-source JSON Schema configuration fields
- Custom `SourceResponseSchema` component with per-status-code response schemas, collapsible property tables, and example payloads
- Build-time data extraction optimizes a 46MB spec down to 26KB for runtime use

## How

### Build pipeline (`prebuild`)
1. `prepare-public-api-spec.js` fetches 17 YAML spec files from `airbyte-platform`, merges them, validates the result, and applies curated tags from `public_api_tags.json` via a hardcoded `OPERATION_TAG_MAPPING` (path+method → display tag). Endpoints not in the mapping are excluded.
2. The script extracts per-endpoint metadata (request params, per-status-code response schemas, examples) into `api-endpoints-dereferenced.json` (26KB) and source connector configurations into `source-configs-dereferenced.json`.
3. The merged spec is written as `public_api_spec.yaml` for the `docusaurus-plugin-openapi-docs` plugin to generate MDX pages.
4. `createApiPageMDGenerator.js` injects `<SourceRequestSchema>` and `<SourceResponseSchema>` components into the generated MDX.
5. `sidebar-generator.ts` loads the autogenerated sidebar and filters it to only categories defined in `public_api_tags.json`.

### Architecture
```
Build time (prebuild):
  fetch 17 YAML specs → merge → validate → apply curated tags → filter endpoints
    ├── Write public_api_spec.yaml (for docusaurus-plugin-openapi-docs)
    ├── Write api-endpoints-dereferenced.json (for custom components)
    ├── Write source-configs-dereferenced.json (for source config component)
    └── Generate MDX pages + sidebar via plugin

Runtime:
  MDX pages render with injected <SourceRequestSchema> / <SourceResponseSchema>
  Components load from api-endpoints-dereferenced.json (26KB)
  Sidebar filtered by public_api_tags.json
```

## Review guide

### Build scripts (`docusaurus/src/scripts/public-api/`)
1. `prepare-public-api-spec.js` — Main build script: fetch, merge, validate, tag mapping, data extraction
2. `constants.js` — Shared path constants
3. `openapi-validator.js` — OpenAPI spec validation
4. `createApiPageMDGenerator.js` — Custom MDX page generator that injects custom components
5. `sidebar-generator.ts` — Sidebar loading and tag-based filtering

### Custom React components (`docusaurus/src/components/`)
6. `SourceRequestSchema/index.tsx` — Request parameters + source configuration selector
7. `SourceResponseSchema/index.tsx` — Per-status-code response schemas with property tables and examples
8. `SourceConfiguration/index.tsx` — Reusable source config component with schema-driven fields
9. `MimeTypePills/index.tsx` — Content type badges

### Configuration
10. `docusaurus/docusaurus.config.ts` — `public-api` plugin config, sidebar generator import
11. `docusaurus/package.json` — `prepare-public-api` script in `prebuild`/`prestart` chains

### Data (source of truth)
12. `docusaurus/src/data/public_api_tags.json` — 15 curated tag categories with descriptions

## User Impact

- Users get a new **Public Airbyte API** section under Developers in the docs sidebar with 15 categories of API endpoints
- Each endpoint page shows request parameters, source-specific configuration schemas (with a searchable dropdown), and per-status-code response details with examples
- No impact on existing docs pages or the Agent Engine API docs
